# RESOURCES.md 格式

此文件定义了针对某个主题的精选可信来源集合。解释说明中的知识应从此处获取，而非来自参数化猜测。

## 模板

```md
# {主题} 资源

## 知识

### {标题}
{链接 | 作者 + 书名}
用途：{何时以及为何使用此资源。}

### {标题}
{链接 | 作者 + 书名}
用途：{何时以及为何使用此资源。}

## 智慧（社区）

### {社区名称}
{链接}
{它擅长什么，以及为什么用户应该考虑加入。}
```

## 规则

1. **仅限高信任来源。** 优先选择一手来源、公认专家、同行评审作品以及具有良好管理机制的社区。排除伪装成教育的营销内容。
2. **每条条目都要有注释。** 裸露的链接是无用的。每条需要一行内容，涵盖它包含什么以及何时使用。
3. **按知识/智慧分组。** 这镜像了 SKILL.md 中的理念。一个资源可以只出现在一个组中。
4. **显式标出缺口。** 如果某个需要覆盖的领域缺乏好的资源，添加一个 `## 缺口` 章节描述缺少什么，以指导未来的搜索。
5. **无情修剪。** 被发现错误、肤浅或偏离任务的资源应被完全移除。五条好来源优于三十条平庸来源。
6. **记录社区偏好。** 如果用户已选择不加入社区，记录此偏好，这样未来的 session 不会继续推荐它们。
