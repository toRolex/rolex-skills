# 学习记录格式

学习记录存储在 `./learning-records/` 中，使用从 0001 开始的递增编号：`0001-<短横线隔开的名称>.md`。目录仅在写入第一条记录时创建。

## 核心理念

学习记录是"教学版本的 ADR（架构决策记录）"。它们捕获非显而易见的课程、关键洞见和已陈述的先前知识，以指导未来的 session 并帮助计算最近发展区。

## 模板

```md
# {编号}: {所学内容的简要描述}

1-3 句话：学到了什么（或建立了什么先前知识），以及它为什么对未来的 session 重要。价值在于记录*这个现在已经知道*以及*它为什么改变接下来要教什么*。
```

## 可选章节

仅在真正有价值时使用：

- **状态** 前置元数据（`active` 或 `superseded by LR-NNNN`）——当先前的理解被替换时有用。
- **证据**——理解是如何展示的（例如，回答的问题、完成的练习）。
- **影响**——哪些未来的 session 被解锁或排除。

## 编号

扫描现有最高编号并加一。

## 何时写入

1. **用户展示了真正的理解**——对非琐碎内容的真正掌握，为接下来教什么设定了新的基准。
2. **用户透露了先前知识**——包括声称的深度，以避免重复教授。
3. **纠正了误解**——高价值：它们能预测相关主题中未来的障碍。
4. **任务因学习而转变**——需要交叉链接到 MISSION.md。

## 不应写入的情况

- 仅仅"覆盖过"的材料
- 已简洁捕获在 GLOSSARY.md 中的内容
- 逐 session 的活动日志。学习记录不是日记。

## 取代

当后来的记录与较早的记录矛盾时，较旧的记录标记为已取代状态，而非删除。理解演变的历史本身就是有用的信号。
