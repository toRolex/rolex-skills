# QA Plan 示例

下面是一份相对完整的 QA plan 示例。它假设最近一批 commit 为在线商城新增了「限时折扣」功能。

## QA Plan: 2026-07-13 — 限时折扣功能

### 测试环境
- 分支：`feature/flash-sale`
- 前置条件：已配置一个正在进行中的限时折扣活动（商品 A 原价 100 元，折扣价 80 元，限购 2 件）

### 测试步骤

#### 1. 活动商品展示
- [ ] **测试什么**：限时折扣商品在活动时间内是否在首页和商品列表中正确显示折扣价与倒计时。
- [ ] **如何测试**：
  1. 打开首页，找到「限时折扣」入口。
  2. 进入商品列表，找到商品 A。
- [ ] **预期结果**：商品 A 显示 80 元折扣价，并显示活动倒计时；原价 100 元以划线价展示。
- [ ] **优先级**：P0

#### 2. 商品详情页折扣信息
- [ ] **测试什么**：商品详情页是否展示折扣信息、限购提示和活动时间。
- [ ] **如何测试**：
  1. 点击商品 A 进入详情页。
  2. 检查价格区域、活动说明和限购数量提示。
- [ ] **预期结果**：页面显示「限时折扣价 80 元」「每人限购 2 件」「活动结束时间：2026-07-13 23:59:59」。
- [ ] **优先级**：P0

#### 3. 加购与数量限制
- [ ] **测试什么**：折扣商品加入购物车时是否受限购数量约束。
- [ ] **如何测试**：
  1. 在商品详情页将数量改为 3，点击「加入购物车」。
  2. 再将数量改为 2，点击「加入购物车」。
- [ ] **预期结果**：数量为 3 时提示「超出限购数量，每人限购 2 件」；数量为 2 时成功加入购物车。
- [ ] **优先级**：P0

#### 4. 购物车价格计算
- [ ] **测试什么**：购物车中折扣商品是否按折扣价计价，小计和总价正确。
- [ ] **如何测试**：
  1. 进入购物车，查看商品 A 的小计。
  2. 继续添加一件普通商品 B（50 元）。
  3. 查看订单总价。
- [ ] **预期结果**：商品 A 小计 160 元（80 × 2），订单总价 210 元。
- [ ] **优先级**：P0

#### 5. 结算页折扣继承
- [ ] **测试什么**：从购物车进入结算页后，折扣价和限购信息保持正确。
- [ ] **如何测试**：
  1. 勾选商品 A 和商品 B，点击「去结算」。
  2. 在结算页核对商品列表、价格和应付总额。
- [ ] **预期结果**：结算页显示商品 A ×2 = 160 元，商品 B ×1 = 50 元，应付总额 210 元。
- [ ] **优先级**：P0

#### 6. 活动开始前/结束后价格
- [ ] **测试什么**：不在活动有效期内的商品是否恢复原价，且不允许以折扣价购买。
- [ ] **如何测试**：
  1. 将系统时间调整到活动开始前（或等待活动结束后），刷新商品详情页。
  2. 尝试以折扣价流程购买。
- [ ] **预期结果**：商品 A 显示原价 100 元；若从旧链接进入折扣结算，提示「活动已结束」并阻止下单。
- [ ] **优先级**：P1

#### 7. 库存不足边界
- [ ] **测试什么**：折扣商品库存不足时是否给出正确提示。
- [ ] **如何测试**：将商品 A 的库存设置为 1，在购物车中数量为 2 时尝试结算。
- [ ] **预期结果**：提示「库存不足，仅剩 1 件」，并阻止提交订单。
- [ ] **优先级**：P1

#### 8. 多端一致性（可选）
- [ ] **测试什么**：H5/小程序/APP 中的折扣展示和购买流程与 Web 一致。
- [ ] **如何测试**：在 H5/小程序/APP 中重复步骤 1–5 的核心路径。
- [ ] **预期结果**：价格、限购、倒计时、结算金额与 Web 端一致。
- [ ] **优先级**：P2

### 统计
- P0：5 项
- P1：2 项
- P2：1 项
